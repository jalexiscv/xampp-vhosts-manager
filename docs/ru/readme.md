## Описание

Xampp vHosts Manager — это инструмент, упрощающий создание и управление виртуальными хостами в Xampp для Windows. Он позволяет:

- Создавать виртуальные хосты за секунды
- Генерировать самоподписанные SSL-сертификаты без использования сложных команд OpenSSL
- Управлять всем из консоли (`xvhost`) или через интерактивную веб-панель

Инструмент предназначен для разработчиков, работающих с Xampp, которым необходимо быстро и легко создавать несколько локальных доменов с SSL.

---

## Возможности

- Создание виртуальных хостов с произвольным именем
- Удаление существующих виртуальных хостов
- Отображение подробной информации о хосте
- Список всех виртуальных хостов
- Добавление самоподписанных SSL-сертификатов к любому хосту
- Удаление SSL-сертификатов с хоста
- Изменение Document Root существующего хоста
- Запуск, остановка и перезапуск Apache
- **Веб-панель** с современным тёмным графическим интерфейсом
- Собственный CA-сертификат, который можно установить как доверенный центр в Windows

---

## Совместимость

- Совместим со всеми версиями Xampp, использующими PHP 5.4 или выше
- **Не** поддерживает портативную версию Xampp
- Операционная система: только Windows

---

## Требования

1. **Установленный Xampp** (разумеется)
2. **PHP** — доступный; установщик автоматически добавляет путь к PHP из Xampp в системную переменную PATH
3. **Опционально:** Установленный Composer (только если вы хотите устанавливать через Composer)

## Установка

### Ручная загрузка (рекомендуется)

1. Скачайте [последнюю версию](https://github.com/jalexiscv/xampp-vhosts-manager/releases/latest)
2. Распакуйте содержимое в общую папку, например `C:\xampp\hosts` или `D:\xvhm`
   > **Важно:** Не размещайте в `C:\Program Files` или в местах, требующих прав Администратора для записи файлов
3. Откройте терминал **от имени Администратора**
4. Перейдите в каталог, куда вы распаковали менеджер:
   ```cmd
   cd /D C:\xampp\hosts
   ```
5. Запустите установщик:
   ```cmd
   xvhost install
   ```
6. Следуйте инструкциям на экране (путь к Xampp, предполагаемый путь DocumentRoot и т.д.)
7. Закройте терминал (для очистки временных переменных окружения)
8. Откройте новый терминал — теперь вы можете использовать `xvhost` из любого места

### Установка менеджера

Команда `xvhost install` автоматически выполняет:

- Регистрирует путь менеджера в системной переменной PATH Windows
- Настраивает файл `settings.ini` с путями вашего Xampp
- Генерирует сертификат Центра сертификации (CA) в `cacert/`
- Предоставляет необходимые разрешения для файла `hosts` Windows

---

## Обновление

1. Сделайте резервную копию файла `settings.ini` и папки `cacert/`
2. Удалите всё содержимое каталога менеджера
3. Скачайте [последнюю версию](https://github.com/jalexiscv/xampp-vhosts-manager/releases/latest) и распакуйте её в тот же каталог
4. Восстановите файл `settings.ini` и папку `cacert/`

## Использование — Консоль (xvhost)

После установки вы можете выполнять `xvhost` из любого терминала **без прав Администратора** (за исключением команд, изменяющих системную конфигурацию).

### Справка

```
xvhost help
```

### Создание виртуального хоста

```
xvhost new [ИМЯ_ХОСТА]
```

Пример:
```
xvhost new mistio.local
```

> Параметр `ИМЯ_ХОСТА` необязателен. Если его не указать, система запросит его в процессе.

Во время создания вас спросят:
- **Document Root** — путь, где будут находиться файлы сайта
- **Email администратора**
- **Добавить SSL?** — если ответить утвердительно, будет автоматически сгенерирован самоподписанный SSL-сертификат

### Просмотр информации о хосте

```
xvhost show [ИМЯ_ХОСТА]
```

Пример:
```
xvhost show mistio.local
```

### Список всех хостов

```
xvhost list
```

### Удаление хоста

```
xvhost remove [ИМЯ_ХОСТА]
```

Пример:
```
xvhost remove mistio.local
```

Эта команда удаляет конфигурацию Apache, SSL (если существует) и запись из файла `hosts` Windows.

### Добавление SSL-сертификата

```
xvhost add_ssl [ИМЯ_ХОСТА]
```

Пример:
```
xvhost add_ssl mistio.local
```

### Удаление SSL-сертификата

```
xvhost remove_ssl [ИМЯ_ХОСТА]
```

### Изменение Document Root

```
xvhost change_docroot [ИМЯ_ХОСТА]
```

### Остановка/Запуск/Перезапуск Apache

```
xvhost stop_apache
xvhost start_apache
xvhost restart_apache
```

### Регистрация пути приложения

Если вы переместили менеджер в другую папку, зарегистрируйте новый путь командой:

```
xvhost register_path
```

> Требует прав Администратора.

### Разрешения для файла hosts в Windows

Чтобы менеджер мог автоматически добавлять записи в файл `hosts` Windows:

```
xvhost grantperms_winhosts
```

> Требует прав Администратора. Выполнить нужно только один раз.

## Веб-панель (графический интерфейс)

Помимо консоли, проект включает **современную веб-панель**, доступную через браузер.

### Требования веб-панели

- Запущенный Apache с PHP (очевидно)
- Каталог менеджера должен находиться в дереве документов Apache, либо должен быть виртуальный хост, указывающий на него

### Как получить доступ

Если вы установили менеджер в `C:\xampp\hosts`, вы можете:

**Вариант A — Виртуальный хост:** Создайте виртуальный хост, указывающий на каталог:
```
xvhost new xvhm.local
```
Когда будет запрошен Document Root, укажите `C:\xampp\hosts` и откройте `http://xvhm.local`

**Вариант B — Напрямую через PHP:** Откройте терминал и выполните:
```
C:\xampp\php\php.exe -S localhost:8080 -t C:\xampp\hosts
```
Затем откройте `http://localhost:8080`

### Функции панели

Веб-панель позволяет:

- **Просматривать все виртуальные хосты** с их статусом SSL
- **Создавать новые хосты** через форму (имя, Document Root, email, SSL)
- **Открывать хосты** в браузере одним кликом
- **Удалять хосты** напрямую
- **Управлять Apache** (запуск, остановка, перезапуск)
- **Видеть состояние Apache** в реальном времени
- Интерфейс с тёмным дизайном (dark mode)

## SSL-сертификаты

### Как это работает?

Менеджер работает с двумя типами сертификатов:

1. **Корневой CA-сертификат** (`cacert/cacert.crt`) — генерируется один раз при установке. Это Центр сертификации, подписывающий все сертификаты виртуальных хостов.

2. **Сертификаты хостов** — каждый хост с включённым SSL получает собственный сертификат, подписанный корневым CA. Сертификаты включают Subject Alternative Names (SANs) для имени хоста и `www.*`.

**Расположение:**
- Сертификаты CA: `C:\xampp\hosts\cacert\`
- Сертификаты хостов: `C:\xampp\apache\conf\extra\certs\`
- Приватные ключи: `C:\xampp\apache\conf\extra\keys\`
- SSL-конфигурации Apache: `C:\xampp\apache\conf\extra\vhosts_ssl\`

### Установка доверия к CA-сертификату в Windows

Чтобы браузер **не показывал предупреждения безопасности** при открытии `https://вашсайт.local`:

1. Откройте терминал **от имени Администратора**
2. Выполните:
   ```cmd
   certutil -addstore -f Root C:\xampp\hosts\cacert\cacert.crt
   ```
   (скорректируйте путь, если установили менеджер в другую папку)
3. Перезапустите браузер

Если вы используете **Chrome или Edge**, также можно:
- Перейти в `chrome://settings/security` → Управление сертификатами → Импорт
- Выбрать `C:\xampp\hosts\cacert\cacert.crt`
- Выбрать «Доверенные корневые центры сертификации»
- Завершить

### Установка доверия к CA-сертификату в Firefox

Firefox использует собственное хранилище сертификатов, независимое от Windows:

1. Откройте Firefox
2. Перейдите в **Настройки** → **Конфиденциальность и безопасность** → **Сертификаты**
3. Нажмите **Просмотр сертификатов** → вкладка **Центры**
4. Нажмите **Импорт**
5. Выберите `C:\xampp\hosts\cacert\cacert.crt`
6. Отметьте **«Доверять этому центру сертификации для идентификации веб-сайтов»**
7. Нажмите ОК

## Конфигурация

Вся конфигурация находится в файле `settings.ini` в каталоге менеджера:

```ini
[DirectoryPaths]
; Путь к вашей установке Xampp
Xampp = "C:\xampp"

[Suggestions]
; Предполагаемый путь DocumentRoot при создании нового хоста
; {{host_name}} заменяется на имя хоста
DocumentRoot = "C:\xampp\hosts\{{host_name}}"

; Предполагаемый email для ServerAdmin
AdminEmail = "admin@localhost"

[ListViewMode]
; Количество записей, отображаемых на странице при выводе списка хостов
RecordPerPage = "5"
```

## Решение проблем

### SSL не генерируется

**Симптом:** При выполнении `xvhost add_ssl mistio.local` процесс завершается ошибкой без создания сертификата.

**Причина:** В более старых версиях серийный номер сертификата генерировался через `php -r "echo md5(...)"`, но `php` не был в системном PATH. Текущая версия исправляет это, используя `openssl rand -hex 16`.

**Решение:** Убедитесь, что у вас установлена последняя версия менеджера. Если проблема сохраняется, проверьте, существует ли `C:\xampp\apache\bin\openssl.exe`.

### Браузер показывает предупреждение безопасности

**Симптом:** При открытии `https://mistio.local` отображается «Ваше подключение не защищено» (NET::ERR_CERT_AUTHORITY_INVALID).

**Причина:** SSL-сертификат подписан локальным CA, но Windows/Firefox не доверяет ему.

**Решение:** Установите CA-сертификат как доверенный центр (см. раздел [Установка доверия к CA-сертификату в Windows](#установка-доверия-к-ca-сертификату-в-windows)).

### Не удаётся разрешить имя хоста

**Симптом:** `http://mistio.local` не загружается, «Не удалось найти сервер».

**Причина:** Хост отсутствует в файле `hosts` Windows.

**Решение:** Убедитесь, что вы выполнили `xvhost grantperms_winhosts` от имени Администратора (только один раз). Затем создайте хост через `xvhost new mistio.local` или добавьте вручную:
```
127.0.0.1   mistio.local
127.0.0.1   www.mistio.local
```
в файле `C:\Windows\System32\drivers\etc\hosts`

### Apache не запускается

**Симптом:** Apache не запускается после установки или изменения хоста.

**Возможные причины и решения:**
- **Порт занят:** Другая программа (Skype, IIS, Docker) использует порт 80 или 443. Остановите эту программу или измените порты в `C:\xampp\apache\conf\httpd.conf`
- **Ошибка синтаксиса:** Проверьте командой:
  ```cmd
  C:\xampp\apache\bin\httpd.exe -t
  ```
- **Отсутствует OpenSSL:** Убедитесь, что `C:\xampp\apache\bin\openssl.exe` существует
- **SSL-конфликты:** Если есть повреждённые SSL-конфигурации, проверьте файлы в `C:\xampp\apache\conf\extra\vhosts_ssl\`

### Ошибка прав доступа при создании хоста

**Симптом:** «Permission denied» при создании хоста или записи в файл `hosts`.

**Решение:** Выполните один раз:
```
xvhost grantperms_winhosts
```
от имени Администратора. После этого повышенные права для создания хостов больше не понадобятся.
